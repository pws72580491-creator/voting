🛠️ 배포 단계별 가이드

  • 1

    GitHub 계정 만들기 (이미 있으면 건너뛰기)

    github.com/signup 에서 무료 계정 생성

  • 2

    새 저장소(Repository) 생성

    • github.com/new 접속
    • Repository name: voting (원하는 이름 가능)
    • Public 선택 (GitHub Pages 무료 사용 조건)
    • ✅ Add a README file 체크
    • Create repository 클릭
  • 3

    파일 업로드

    • 저장소 메인 페이지 → Add file → Upload files
    • 아래 파일들을 드래그&드롭:
      index.html
      voting-guide.html
      README.md
    • 하단 Commit changes 클릭
    ⚠️ voting-system.html이 아니라 index.html로 업로드해야 합니다!
  • 4

    GitHub Pages 활성화

    • 저장소 상단 Settings 탭 클릭
    • 왼쪽 메뉴 → Pages
    • Source → Deploy from a branch
    • Branch → main / root 선택 → Save
    Settings → Pages
    Source: Deploy from a branch ✓
    Branch: main / / (root)  Save
  • 5

    배포 완료 — URL 확인

    1~2분 후 Settings → Pages 상단에 URL이 표시됩니다:

    🌐 배포 주소
    https://내-아이디.github.io/voting/

    이 URL을 참여자에게 카카오톡, 문자, QR코드 등으로 공유하면 됩니다.

🔥 Firebase 규칙 확인

외부에서 접근 허용 (이미 설정된 경우 건너뛰기)

Firebase Console → voting-9aa78 → Realtime Database → 규칙(Rules)

{ "rules": { ".read": true, ".write": true } }

위 규칙으로 설정 후 게시(Publish) 클릭

⚠️ 이 규칙은 누구나 읽고 쓸 수 있습니다. 소규모 행사·내부 투표에 적합하며, 대규모 공개 투표에는 인증 규칙 추가를 권장합니다.

📲 참여자 공유 방법

방법내용
URL 직접 공유 카카오톡, 슬랙, 문자로 주소 전송
QR코드 생성 qr.io 또는 qr-code-generator.com에서 URL 입력 → QR 이미지 다운로드
단축 URL bit.ly로 짧은 주소 만들기
✅ 참여자는 인터넷이 되는 기기 + 브라우저만 있으면 됩니다. 앱 설치 불필요.

🔄 파일 수정 후 재배포

  1. GitHub 저장소 접속 → 수정할 파일 클릭
  2. 우측 상단 연필(✏️) 아이콘 클릭
  3. 내용 수정 후 Commit changes
  4. 1~2분 후 자동 재배포 완료
💡 또는 저장소에서 Add file → Upload files로 파일 덮어쓰기 가능
