⚠️ Firebase 보안 규칙 확인
Firebase Console → 규칙(Rules) 탭에서 아래 규칙이 설정되어 있는지 확인하세요.
{ "rules": { ".read": true, ".write": true } }
이 규칙은 테스트 / 소규모 행사 용도입니다. 외부 공개 서비스에는 인증 규칙을 추가하세요.

💻 여러 기기에서 동시 사용

동일한 voting-system.html 파일을 각 기기에서 열고 같은 Firebase URL로 연결하면 됩니다.

역할사용 탭권장 기기
관리자 PC ⚙️ 관리자 탭 (비밀번호 로그인) 노트북 / 데스크탑
투표 기기 🗳 투표하기 탭 태블릿 / 노트북
결과 모니터 📊 실시간 결과 탭 대형 모니터 / TV
✅ 인터넷만 연결되어 있으면 어디서든 같은 데이터를 공유합니다.

⚠️ 주의사항

  • 중복 투표 방지는 유권자 ID 기반입니다. 같은 ID를 두 번 입력하면 차단됩니다.
  • 투표 중 후보자를 삭제하면 이미 해당 후보에게 투표된 표도 결과에서 사라집니다.
  • 투표 초기화는 되돌릴 수 없습니다. 결과를 먼저 스크린샷으로 저장하세요.
  • Firebase 무료 플랜(Spark)은 동시 연결 100개, 월 10GB 전송 제한이 있습니다. 소규모 행사에는 충분합니다.
  • 현재 Firebase 규칙이 공개 읽기/쓰기로 설정되어 있습니다. 중요한 선거는 인증 규칙을 추가하세요.

🔧 자주 발생하는 문제

❌ "연결 실패" 오류가 납니다
  • Firebase Console에서 Realtime Database가 생성되어 있는지 확인
  • 규칙(Rules)이 "read": true, "write": true로 설정되어 있는지 확인
  • URL 끝에 슬래시(/)가 없는지 확인
❌ 투표 버튼이 비활성화됩니다
  • 관리자 탭에서 🟢 투표 시작하기 버튼을 눌렀는지 확인
  • 유권자 ID와 후보자를 모두 선택했는지 확인
❌ 결과가 갱신되지 않습니다
  • 인터넷 연결 상태를 확인하세요
  • 3초 주기 자동 갱신 — 잠시 기다려보세요
  • 페이지를 새로고침(F5)하면 재연결됩니다
